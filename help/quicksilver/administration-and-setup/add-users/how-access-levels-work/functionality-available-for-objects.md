## リソース管理

Standard ライセンスを持つユーザーのみが、リソース管理に対するフルアクセス権を持つことができます。 その他のライセンスタイプでは、Workfrontのリソース管理に対するアクセスを制限したり、アクセスできなくしたりできます。

| アクション | 標準 | ライト | コントリビューター | 外部ユーザー |
|----------------------------------------------------------------|----------|--------|-------------|---------------|

| 資源プールの作成、編集、削除&#42;&#42; | ✓&#42; |   |   |   |
| `Update Planned Hours in the Workload Balancer`&#42;&#42;&#42; | ✓&#42; |   |   |   |
| リソースプランナーのプロジェクト優先度の表示 | ✓&#42; |   |   |   |
| リソース計画ツールでのリソース割り当ての表示 | ✓&#42; | ✓&#42; |   |   |
| リソースプールの表示 | ✓&#42; | ✓&#42; |   |   |
| リソース計画ツールの予算リソース&#42;&#42; | ✓ |   |   |   |
| プロジェクト、テンプレート、ユーザーに資源プールを添付する | ✓ |   |   |   |

{style="table-layout:auto"}

&#42; アクセスレベルを使用して、Workfront管理者はこの機能を無効または有効にできます。

&#42;&#42;財務データへの追加のアクセス権と、プロジェクトの財政面での権限が必要です。 財務データへのアクセス権を持たないプランナ・ユーザーに資源管理アクセス権を付与した場合、ユーザーは「生産資源プランナ」で時間別割当を表示できますが、原価ビューに切り替えたり、ビジネス・ケースを表示することはできません。 詳しくは、 [財務データへのアクセス権の付与](../../../administration-and-setup/add-users/configure-and-grant-access/grant-access-financial.md) および [オブジェクトに対する財務権限の共有](../../../workfront-basics/grant-and-request-access-to-objects/share-financial-permissions-object.md).

&#42;&#42;&#42;[ 詳細設定 ] で [ 割り当てを行う ] が有効な状態で、オブジェクトに Contribute する権限が必要です。 詳しくは、 [継承された権限とオブジェクトの階層について](../../../workfront-basics/grant-and-request-access-to-objects/sharing-permissions-on-objects-overview.md#sharing-an-object) 記事内 [オブジェクトに対する共有権限の概要](../../../workfront-basics/grant-and-request-access-to-objects/sharing-permissions-on-objects-overview.md).
