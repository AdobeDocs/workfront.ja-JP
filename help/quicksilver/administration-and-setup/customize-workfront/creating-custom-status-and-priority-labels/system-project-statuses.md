# システムプロジェクトのステータスの概要

<!--Audited: 12/2023-->

Workfront には、組み込みのシステムプロジェクトステータスが 9 つあります。

以下の表の最初の 3 つは必須です。つまり、それらのロックの解除、名前の変更、並べ替えは可能ですが、非表示にしたり削除したりすることはできません。

通常、プロジェクトステータスの変更は手動で行われます。ただし、システムで発生している他のアクティビティに応じて、プロジェクトのステータスが自動的に変更された場合、次のリストで概要を示すシナリオがいくつかあります。

Workfrontは、Adobe Workfrontインスタンスに次のプロジェクトステータスを提供します。

   <td> <p> プロジェクトのすべてのタスクと問題が完了し、プロジェクトが完了します。</p> 
     <p>プロジェクトの完了モードが手動に設定されている場合、プロジェクトマネージャーがこのステータスを手動で選択し、プロジェクトチームのユーザーに対して、プロジェクトの作業を止めるよう通知します。</p> 
    <p>プロジェクトの完了モードが自動に設定されている場合、プロジェクトのすべてのタスクとイシューが完了とマークされたときに、Workfront がプロジェクトを自動的に完了とマークします。 
    <p><b>重要</b> </p>
    <p>プロジェクトに対するすべてのタスク、問題および承認が解決された場合にのみ、プロジェクトを完了としてマークできます。</p> </td> 
   <td>
    <p>プロジェクトチームのユーザーは、デフォルトでは（カスタムフィルターなしで）Workfrontの「プロジェクト」領域に、プロジェクトリストのプロジェクトを表示できません。 プロジェクト上で割り当てられたタスクとイシューには、作業リクエストリストや作業中リストが表示されません。 </p>
    <p>プロジェクトに関連するすべての通知（ステータス変更通知を除く）は、プロジェクトチームのユーザーに送信されなくなりました。</p>
    <p>プロジェクトのタイムラインは、システムでは計算されなくなりました。 </p>
    <p>プロジェクトをコピーできません。</p>
    <p>プロジェクトが完了とマークされている間、ユーザーが追加のアクションを実行しないようにできます。 </p><p>完了とマークされたプロジェクトに対するアクションを制限する方法について詳しくは、<a href="../../../administration-and-setup/set-up-workfront/configure-system-defaults/set-project-preferences.md" class="MCXref xref">システム全体のプロジェクト環境を設定</a>を参照してください。</p></td>
