# システムプロジェクトのステータス

Workfrontには、9 つの組み込みのシステムプロジェクトステータスがあります。

以下の表の最初の 3 つは必須です。つまり、ロックの解除、名前の変更、並べ替えは可能ですが、非表示や削除はできません。

通常、プロジェクトのステータスの変更は手動でおこなわれます。 ただし、システムで発生している他の要因に応じて、プロジェクトのステータスが自動的に変更された場合、次のリストで概要を示すシナリオがいくつかあります。

Workfrontインスタンスには、次のプロジェクトステータスが用意されています。

   <td>計画（必須ステータス）</td> 
   <td> <p>プロジェクトマネージャーは、プロジェクトのタイムライン、タスクの割り当て、および承認を計画しています。 プロジェクトマネージャは、このステータスを手動でプロジェクトに設定します。</p> <p><b>ヒント</p> <p> Workfrontの新規プロジェクトのデフォルトのステータスは、Planning に設定することをお勧めします。 Workfrontの管理者は、プロジェクト環境設定の「プロジェクト」領域で、すべての新規プロジェクトのデフォルトステータスを変更できます。</p> </td>

<p> Workfrontの管理者は、プロジェクト環境設定の「プロジェクト」領域で、新規プロジェクトのデフォルトのステータスを変更できます。 詳しくは、 <a href="../../../administration-and-setup/set-up-workfront/configure-system-defaults/set-project-preferences.md" class="MCXref xref">システム全体のプロジェクト環境設定の指定</a>.</p> </td> 
   <td> <p>プロジェクトチームのユーザーは、デフォルト（カスタムフィルターなし）で、Workfrontの「プロジェクト」領域にプロジェクトを「プロジェクト」リストに表示できます。 プロジェクト上で割り当てられたタスクと問題が、作業リストに表示されます。 タスクや問題に関する作業を受け入れ、作業リストに移動させることができます。</p> <p>また、現在のプロジェクトでは、タイムラインの変更、割り当て、必要なアクション、承認に関するすべての通知が、プロジェクトチームのユーザーに送信されます。</p> <p>プロジェクトの [ 更新の種類 ] が [ 自動 ]、[ 変更時 ]、または [ 自動 ] と [ 変更時 ] に設定されている場合、プロジェクトのタイムラインは自動的に計算されます。</p> <p>ヒント：プロジェクトがこのステータスの場合、ユーザーが受け取る通知が多くならないように、プロジェクト計画の調整を最小限に抑えることをお勧めします。</p> </td> 
  </tr>

   <td> <p> プロジェクトが完了します。</p> 
     <p>プロジェクトの [ 完了モード ] が [ 手動 ] に設定されている場合、プロジェクトマネージャはこの状態を手動で選択し、プロジェクトチームのユーザーに対して、プロジェクトの作業を中止するよう通知します。</p> 
    <p>プロジェクトの「完了モード」が「自動」に設定されている場合、Workfrontは、プロジェクトのすべてのタスクと問題が「完了」とマークされたときに、プロジェクトを自動的に「完了」とマークします。 
    <p><b>重要</b> </p>
    <p>プロジェクトに対するすべてのタスク、問題および承認が解決された場合にのみ、プロジェクトを完了としてマークできます。</p> </td> 
   <td>
    <p>プロジェクトチームのユーザーは、デフォルトでは（カスタムフィルターなしで）Workfrontの「プロジェクト」領域に、プロジェクトリストのプロジェクトを表示できません。 プロジェクト上で割り当てられたタスクとタスクには、作業要求リストや作業中リストが表示されません。 </p>
    <p>プロジェクトに関連するすべての通知（状態変更通知を除く）は、プロジェクトチームのユーザーに送信されなくなります。</p>
    <p>プロジェクトのタイムラインは、システムではもはや計算されません。 </p>
    <p>プロジェクトをコピーできません。</p>
    <p>プロジェクトが「完了」とマークされている場合に、ユーザーが追加のアクションを実行しないようにできます。 </p><p>完了とマークされたプロジェクトに対するアクションを制限する方法の詳細については、「 <a href="../../../administration-and-setup/set-up-workfront/configure-system-defaults/set-project-preferences.md" class="MCXref xref">システム全体のプロジェクト環境設定の指定</a>.</p></td>

   <td> <p>プロジェクトチームのユーザーは、デフォルトでは（カスタムフィルターなしで）Workfrontの「プロジェクト」領域に、プロジェクトリストのプロジェクトを表示できません。 プロジェクト上で割り当てられたタスクとタスクは、作業リストに表示されなくなります。 </p> <p>承認の決定は、タスクやタスクに与えることはできません。</p> <p>タイムラインの変更、割り当て、必要なアクション、承認に関する通知は、プロジェクトチームのユーザーには送信されません。</p> <p> <p><b>注意</b>：プロジェクトを保留中にすると、プロジェクトのタイムラインは停止しません。 プロジェクトで積極的に作業している人がいなくても、プロジェクトは「危険な状態」または「問題が発生」と表示されます。 プロジェクトを再度現在のタスクに戻すと、残りのオープンタスクの日付を手動で調整する必要が生じる場合があります。そのため、プロジェクトで更新された進捗を表示できます。</p> </p> </td>
