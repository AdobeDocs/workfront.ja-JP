# 環境のプロモーションに関するトラブルシューティング

この記事では、環境プロモーションのトラブルシューティング方法について説明します。

## 問題：環境プロモーションパッケージが停止または失敗しています

環境プロモーションパッケージが停止または失敗している場合は、次の手順を実行します。

* パッケージのインストールが10 ～ 15分後に停止した場合、またはパッケージのインストールに失敗した場合は、既存のパッケージを再構築するか、新しいパッケージを作成します。

* パッケージのインストールに失敗した場合は、1つ以上のオブジェクトに問題がある可能性があります。 エラーメッセージを確認します。これにより、オブジェクトを特定し、問題の特定に役立ちます。 オブジェクトの問題に対処したら、パッケージを再構築してインストールを再試行します。

* インストールで問題が発生する場合は、別のターゲット環境でインストールをレプリケートしてみてください。 選択したオブジェクトやインストールアクションを含め、元のインストールにできる限り近づけます。

* パッケージを組み立てた後は、パッケージの内容を必ず確認して、期待するオブジェクトが含まれていることを確認することをお勧めします。


## 問題：カスタムフォームの昇格に失敗する

これは、カスタムフォームに計算フィールドが含まれているために発生する可能性があります。 計算フィールドがカスタムフォームで参照されていないフィールドを参照する場合、このフォームを含むパッケージは昇格せず、ユーザーには次のメッセージが表示される場合があります。

「次のフィールドは無効です：無効なカスタム式が無効な式：無効なカスタム式」

この問題は、ターゲット環境のカスタムフォームに添付されていない既存のフィールドを参照する場合や、新しく作成されたフィールドを使用する場合に発生する可能性があります。

この問題を解決するには、次のいずれかの操作を行います。

* 参照フィールドを含むプロジェクトタイプのカスタムフォームを使用して、パッケージを作成します。 このパッケージをターゲット環境に昇格した後、計算フィールドを含む最初に意図したパッケージを昇格します。
* 参照先のフィールドをターゲット環境で手動で作成し、それをプロジェクトタイプのカスタムフォームに関連付けます。 これが完了すると、フィールドが認識され、エラーが発生することなくパッケージを昇格できます。
