# Data Connectのジョブ履歴ビューの使用

ジョブ履歴ビューでは、Workfront管理者は各データリフレッシュジョブの詳細レコードにアクセスできます。 これらのレコードは、データを最新の状態に保ち、プロセスを実行し、ビジネスビジュアライゼーションを更新する際に理想的なタイムラインを確立するのに役立つ、貴重なinsightをジョブに提供します。

![ ジョブ履歴ビュー](assets/job-history-overview.png)

ジョブ履歴ビューの列には、次の情報が含まれます。

* **OBJECT_NAME**: ジョブに関連付けられているオブジェクトの名前を表示します。
* **SCHEDULED_TIME**: ジョブの開始時間を表示します。
* **COMPLETED_TIME**: ジョブの完了時間を表示します。
* **LATEST_FLAG**: ジョブが最新の更新の一部であったかどうかを示します。
* **状態**: ジョブのステータスを表示します。 詳しくは、この記事の次の節を参照してください。[使用可能なジョブステータス ](#available-job-statuses)。
* **LAST_UPDATED**: ジョブの最後に更新されたタイムスタンプ。

>[!NOTE]
>
>ジョブ履歴ビューには、スケジュールされたジョブの過去72時間の詳細が含まれます。


## 使用可能なジョブのステータス

各データ接続ジョブには、成功、スキップ、または失敗したかどうかを示すステータスが割り当てられます。

<table>
    <tr>
        <td><b>ジョブステータス</b></td>
        <td><b>定義</b></td>
    </tr>
    <tr>
        <td>成功</td>
        <td>ジョブは利用可能な各更新を正常に処理し、そのレコードタイプのすべての更新がデータレイクに反映されるようになりました。</td>
    </tr>
    <tr>
        <td>スキップ</td>
        <td>レコードの種類に対して処理する更新がキューにないため、ジョブはスキップされました。</td>
    </tr>
    <tr>
        <td>失敗</td>
        <td>ジョブを実行できませんでした。 このような場合、キュー内のデータがデータレイクにコミットされていない可能性があります。 キューに残っているレコードは、そのレコードタイプの次のスケジュール済みジョブで処理されます。 </td>
    </tr>
   </table>


## ジョブの実行とロギング動作に関する考慮事項

Snowflakeでは、ジョブの実行が処理され、ジョブ履歴ビューに記録される方法に影響を与える可能性があるジョブスケジューラー最適化ツールを使用しています。 このログ動作は、処理するデータがあるかどうかに応じて異なる場合があります。

例えば、特定のオブジェクトに対して処理する新しい行がない場合、次のいずれかの結果が発生する可能性があります。

* **ジョブが実行され、スキップ済みとしてマークされます**: Snowflakeは、処理する行がないことを検出し、ジョブを実行し、スキップ済みステータスをテーブルに記録します。

* **ジョブが実行されません**: Snowflakeは、処理する行がないと判断し、ジョブを実行せず、スキップされたステータスをテーブルに記録します。

  >[!NOTE]
  >
  >ジョブが実行されない2番目のシナリオでは、そのオブジェクトの最新のレコードは、想定されるスケジュールと一致しないタイムスタンプを反映している場合があります。

つまり、行が処理されなくてもジョブは実行されたと見なされ、特定のジョブに対するジョブスケジューラーの動作に応じてログに記録される場合とされない場合があります。
