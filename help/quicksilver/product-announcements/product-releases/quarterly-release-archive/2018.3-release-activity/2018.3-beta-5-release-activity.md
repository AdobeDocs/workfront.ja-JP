## ホームエリアで提出した承認を追跡 {#track-approvals-you-ve-submitted-in-the-home-area}

これで、「ホーム」領域を使用して、他のユーザーに送信した承認を操作できるようになりました。

自宅から、次の操作を実行できます。

* 承認の取り消し
* ユーザーに承認を通知
* ドキュメント承認にユーザーを追加
* 配達確認の承認を表示

   以前は、「自分の作業」領域でのみ、送信済みの承認を操作できました。 

詳しくは、 [承認を管理](../../../../review-and-approve-work/manage-approvals/manage-approvals.md). 

## WorkfrontのFlashベースのツールの代わり {#replacement-of-flash-based-tools-in-workfront}

Flashテクノロジーに依存するすべての機能は、2018.3 リリースでWorkfrontから削除される予定で、新しいソリューションに置き換えられる予定です。

これらの機能の削除によって影響を受けるすべての領域と、それらに置き換わるツールの詳細については、 [Adobe WorkfrontのFlashベースのツールの代わり](../../../../product-announcements/announcements/announcement-archive/replace-flash-tools.md).

## 作業用ライセンス・ユーザーがプロジェクトの「計画」サブタブにアクセスできるようにする {#allowing-work-license-users-to-access-the-scheduling-subtab-of-a-project}

作業用ライセンスを持つユーザーは、「スタッフ」タブで、プロジェクトの「スケジュール設定」サブタブにアクセスできるようになりました。

この機能強化以前は、プランライセンスを持つユーザーのみがこのサブタブにアクセスできます。

プロジェクトの「スケジュール設定」サブタブへのアクセスに必要なアクセスについては、 [リソーススケジュールの概要](../../../../resource-mgmt/resource-scheduling/get-started-resource-scheduling.md) in [リソーススケジュールの概要](../../../../resource-mgmt/resource-scheduling/get-started-resource-scheduling.md).

## リソースプランナーのフィルターの改善 {#improved-filters-in-the-resource-planner}

リソースプランナーの情報をフィルタリングして、カスタムフィールドを含む、プロジェクト固有のフィールドとユーザー固有のフィールドを含めることができるようになりました。

この改善がおこなわれる前は、リソースプランナーの組み込みフィールドと条件の数に制限がある場合にのみフィルタリングできました。

リソース・プランナの情報のフィルタリングの詳細は、 [リソースプランナーの情報のフィルタリング](../../../../resource-mgmt/resource-planning/filter-resource-planner.md).

## リソース・プランナでの実績時間の表示 {#view-actual-hours-in-the-resource-planner}

実績時間は、リソース・プランナのユーザー・ビューで表示できます。

この改善以前は、実績時間はリソース・プランナには表示されませんでした。

リソース・プランナでの時間情報の表示の詳細は、次を参照してください： [リソースプランナーの概要](../../../../resource-mgmt/resource-planning/get-started-resource-planner.md).

## ビジネス事例エクスポートのリソースプランナーデータ {#resource-planner-data-in-the-business-case-export}

ビジネス事例をPDFにエクスポートする場合、エクスポートされたファイルの財務情報に、「生産資源予算」領域と「生産資源プランナ」に情報が反映されるようになりました。 

以前は、書き出されたファイルは、「レガシー生産資源見積」領域と「生産能力プランナー」領域の情報を反映していました。 

ビジネスケースのエクスポートの詳細については、 [プロジェクトのビジネスケースの作成](../../../../manage-work/projects/define-a-business-case/create-business-case.md) セクション [プロジェクトのビジネスケースの作成](../../../../manage-work/projects/define-a-business-case/create-business-case.md).

## 会社レベルの請求率上書きの有効化 {#enable-company-level-billing-rate-override}

会社レベルの請求率でプロジェクトレベルの請求率を上書きするオプションを有効にできるようになりました。 会社レベルで請求レートを更新した後は、プロジェクト上の財務を手動で再計算して、変更を反映させることができます。

この設定を有効にした場合、会社レベルの請求率は、プロジェクトの過去のジョブの役割率を上書きします。

以前は、会社をプロジェクトから削除してから再び関連付けない限り、会社レベルの請求率の変更をプロジェクトに適用できませんでした。

詳しくは、 [プロジェクト・レベルの請求率を会社レベルの請求率で上書き](../../../../manage-work/projects/project-finances/override-project-level-with-company-level-billing-rates.md). 

## ロールベースの承認の簡略化 {#simplified-role-based-approvals}

役割ベースの承認が、プロジェクトでより簡単に機能するようになりました。 これで、プライマリロールまたはセカンダリロールに割り当てられた承認を表示するには、ユーザーがプロジェクトチームに割り当てられ、ユーザープロファイルにロールが割り当てられている必要があります。

以前は、ユーザーをプロジェクトチームに割り当て、ユーザープロファイルに正しい役割が割り当てられていることを確認し、レガシーチームビルダーでその役割を選択する必要がありました。

詳しくは、 [プロジェクトチームの概要](../../../../manage-work/projects/planning-a-project/project-team-overview.md) 記事内 [プロジェクトチームの概要](../../../../manage-work/projects/planning-a-project/project-team-overview.md).

## SWFファイルが校正ビューアでサポートされなくなりました {#swf-files-no-longer-supported-in-the-proofing-viewer}

AdobeのFlashに関するセキュリティ上の問題が原因で、校正ビューアはSWFファイルから作成された配達確認をサポートしなくなりました。 SWFファイル用に以前に作成した配達確認を開くと、以前に校正ビューアに表示されていた配達確認の内容を示すメッセージが表示されます。 配達確認に対するコメントを表示できます。 また、コメントをダウンロードしたり、元のSWF・ファイルをダウンロードしたりできます。

詳しくは、 [サポートされている検証ファイルの種類とサイズ制限の概要](../../../../review-and-approve-work/proofing/proofing-overview/supported-proofing-file-types.md) 記事内 [サポートされている検証ファイルの種類とサイズ制限の概要](../../../../review-and-approve-work/proofing/proofing-overview/supported-proofing-file-types.md). 

## 校正ビューアのコメントリストでアクセスしやすく {#actions-more-quickly-accessible-in-the-comment-list-in-the-proofing-viewer}

校正ビューアで、コメントに対するアクションをよりすばやく追加または変更できるようになりました。 任意のコメントのその他アイコンをクリックし、ドロップダウンメニューで目的のアクションをクリックします。

以前は、コメントを開き、ドロップダウンメニューを開き、サブメニューを開いて、目的のアクションをクリックする必要がありました。

詳しくは、 [配達確認コメントに対するアクションの使用](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/comment-on-a-proof/use-actions-on-comments-in-viewer.md).

## 校正ビューアでの配達確認の共有の改善 {#proof-sharing-improvement-in-the-proofing-viewer}

校正ビューアから URL を送信すると、表示されるオプションが、「ドキュメント」領域から URL を共有したときに表示されるオプションとまったく同じように整理されます。

以前は、これらのオプションは現在と同じ機能を提供していましたが、ドキュメントリストの対応するオプションとの組織が一致していませんでした。

詳しくは、  [URL を共有](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md#sharing-the-url) 記事内  [校正ビューアから配達確認を共有](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md).

## 校正ビューアでのコメントリストの機能強化 {#comments-list-enhancements-in-the-proofing-viewer}

コメントリストで以下の機能強化がおこなわれました。

* コメントテキストの後ろに薄いグレーの背景が表示され、レビュー担当者名、レビュー時間、配達確認用ページ番号など、コメントパネルに表示される他のテキストとは異なる場所に表示されるようになりました。 これにより、読みたいフィードバックがそれだけの場合に、読み取り専用のフィードバックを簡単にすることができます。

   以前は、コメントパネルの背景全体が白でした。

   詳しくは、を参照してください。

* コメントに対する返信は、コメントの下にインデントされます。 これにより、返信と、その上にある元のコメントとを区別しやすくなります。

   以前は、コメントの下の返信はコメントと左揃えで行われていました。

   詳しくは、のを参照してください。

* 校正ビューアでコメントを解決すると、緑色の解決済みチェックマークアイコンが表示され、コメントの右上隅にテキストが表示されません。 これにより、[ コメント ] パネルの外観がより明確になり、必要な情報を簡単にスキャンできます。

   以前は、コメントの左上隅にある「RESOLVED」という単語の横に「解決済み」チェックマークアイコンが表示されていました 

   詳しくは、のを参照してください。

## 校正ビューアでのビデオの配達確認のズームとパン {#zoom-and-pan-video-proofs-in-the-proofing-viewer}

これで、ビデオの配達確認をズームインし、より詳細に調べる必要のある配達確認の領域にパンできます。

## 校正ビューアでより明るい背景 {#lighter-background-in-the-proofing-viewer}

校正ビューアの背景が少し明るくなりました。 この変更により、黒の背景や境界線を含む校正コンテンツを区別しやすくなりました。

以前は、校正ビューアの背景が黒でした。

## 校正ビューアでコメントが編集された際の視認性の向上 {#better-visibility-when-a-comment-has-been-edited-in-the-proofing-viewer}

レビュー担当者が変更したコメントに、「編集済み」というラベルが表示されるようになりました。 この上にマウスを移動すると、レビュー担当者の名前と変更日が表示されます。

## 校正ビューアの [ コメント一覧 ] で、アクションに関連付けられたコメント数を表示します {#view-the-number-of-comments-associated-with-an-action-in-the-comment-list-in-the-proofing-viewer}

コメントリストでフィルターアイコンをクリックすると、各アクションに関連付けられているコメントの数を確認できるようになりました。

アクションは、組織で有効になっている場合にのみ、コメントリストで使用できます。

## 校正ビューアで配達確認にユーザーを追加する際のステージを指定 {#specify-the-stage-when-adding-users-to-a-proof-in-the-proofing-viewer}

校正ビューアで自動化されたワークフローを持つ配達確認にレビュー担当者を追加する場合は、レビュー担当者が配達確認に対して作業を行うステージを指定できます。 これは、校正ビューア以外でユーザーを配達確認に追加する場合に使用できる機能と同じです。

以前は、校正ビューアから配達確認にレビュー担当者を追加したときに、Workfrontによって、現在アクティブなステージにレビュー担当者が配置されていました。

詳しくは、 [ユーザーを追加して配達確認を共有](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md#sharing-with-individual-users) in [校正ビューアから配達確認を共有](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md).

## インタラクティブまたは静的な Web サイトの配達確認から関連 Web サイトを開く {#opening-the-associated-website-from-an-interactive-or-static-website-proof}

インタラクティブまたは静的な Web コンテンツを含む配達確認を確認する際に、ボタンをクリックすると、校正ビューアから新しいブラウザータブで関連する Web サイトを簡単に開くことができるようになりました。 これにより、2 つの異なるタブを使用して、配達確認に表示される内容を実際の Web サイトと比較できます。

以前は、URL の上にマウスポインターを置いてツールチップをクリックすると、関連する Web サイトを開くことができました。 このアプローチは、アクセスしやすく直感的でした。

## 印刷概要の強化 {#print-summary-enhancements}

「印刷の概要」ページで、次の機能が強化されました。

* 配達確認にすべてのページを含めるか、コメントを含むページのみを含めるかを指定できるようになりました。

   以前は、コメントを含むページのみを含めることができました。

* 配達確認にステージが含まれる場合、配達確認のステージの順序と各ステージに対する決定を示すダイアグラムを含めることができるようになりました。

   以前は、この図は使用できませんでした。

* 複数のマークアップが 1 つのコメントに関連付けられている場合、そのコメントの番号が [ 印刷の概要 ] ページの各マークアップに表示されます。

   以前は、コメント番号は、コメントに関連付けられた最初のマークアップにのみ表示されていました。 数字を含まないコメントの他のマークアップが表示されます。

* 配達確認が Web キャプチャの場合、コメントを含むページには、Web ページがキャプチャされた解像度が表示されます。 これは、例えば、配達確認にモバイルデバイス用にキャプチャされたページと、ノートパソコンおよびデスクトップコンピューター用にキャプチャされた同じページが含まれている場合に便利です。
