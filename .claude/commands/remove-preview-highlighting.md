# プレビューのハイライトを削除（Workfront）

## 範囲

**all**&#x200B;がtrueの場合にのみ適用されます：

1. ユーザーがこのワークフローを呼び出しました（例：**「プレビューのハイライト表示を削除する」**&#x200B;または明らかに同じ意図）。
2. Markdown ファイルのパス **not**&#x200B;に&#x200B;**`product-announcements`**&#x200B;が含まれています（リリースノート、ベータ版、`help/quicksilver/product-announcements/`の通知など、フォルダーツリー全体を除外します）。
3. Markdown ファイルは、以下の&#x200B;**[除外パス](#excluded-paths)**&#x200B;の下の&#x200B;**not**&#x200B;です。
4. Markdown ファイルは、`git log`に、現在のGit ユーザーがユーザーが指定した日付範囲内でプレビューコンテンツ **を追加または変更した**&#x200B;として表示されます（「インベントリの手順」を参照）。
5. 記事には、次の&#x200B;**のうち少なくとも1**&#x200B;が含まれています：
   - Preview-environment **本文の文言または実際のスニペットの段落** （一般的なパターン：「ハイライトされた情報」、「プレビュー環境」、「まだ一般公開されていません」、高速リリースノート） – **not**&#x200B;目次/索引ページの&#x200B;**リンクテキストのみ**&#x200B;からの一致（以下を参照）
   - **`class="preview"`**&#x200B;を持つHTML要素（例：`<span class="preview">`、`<div class="preview">`）
   - **`{{highlighted-preview}}`**&#x200B;や&#x200B;**`{{highlighted-preview-article-level}}`**&#x200B;などのプレビュースニペット **変数**。

範囲が明確でない場合は、編集前に確認する必要があります。

**目次/ インデックスページ – このケースを常にスキップします。** **プレビューに関連する文言が** only **の場合、** inside **でマークダウンリンクの表示テキストが**&#x200B;別の&#x200B;**記事を指している場合、ファイルをインベントリに配置しない&#x200B;*ファイルに**no ***、** no **のスニペット変数、および変数** no **のプレビューのボイラープレートが** リンク外の文書&#x200B;**で表示されます。******`class="preview"`これは、そのページでハイライト表示されるプレビューではありません。目次の説明に過ぎません。 1つのファイルだけでなく、任意のインデックス/目次に適用されます。

### 除外されたパス

ユーザーが明示的にオーバーライドしない限り、これらをインベントリに追加したり、このワークフローで編集したりしないでください。

- `help/quicksilver/documents/adobe-workfront-for-experience-manager-assets-essentials/send-to-aem.md` — プレビューと実稼動環境の同時実行制限は、自動化以外で意図的な編集上の決定が必要です。
- `help/quicksilver/reports-and-dashboards/reports/creating-and-managing-reports/create-manage-reports.md` – 目次を報告します。唯一の「プレビュー」シグナルは、サンドボックス配信のプレビューアーティクルへのリンクです。

## 必要なワークフロー（人間が介在）

承認なしで&#x200B;**not** リポジトリを一括編集してください。

1. **在庫**\
   a. **どの四半期リリース**&#x200B;のプレビューハイライトを削除しているかをユーザーに尋ねます（「2026年第3四半期」または「2026.07」など）。\
   b. **Adobe-wiki MCP ツールを使用して、`https://wiki.corp.adobe.com/spaces/AWF/pages/3631617814/2026+Monthly+Release+Calendar`からリリースカレンダー**&#x200B;を取得します。 見つける：
   - **前**&#x200B;の四半期リリースのうち&#x200B;**実稼動リリース日**→`--since`です。
   - **ターゲット**&#x200B;の四半期リリースの&#x200B;**実稼動リリース日**→`--until`です。
   - 四半期リリースは、「四半期リリース名」列（例：2026.01、2026.04、2026.07、2026.10）で識別されます。
   - **現在の日付が第4四半期（10月～12月）:**の場合、今年のカレンダーを取得した後、ユーザーに来年のリリースカレンダーのURLを指定するように依頼し、必要なすべての四半期生産日を取得できるようにします。
c.現在のGit ユーザーを決定し、手順bの実稼動リリース日を使用して次を実行します。

   ```bash
   GIT_USER=$(git config user.name)
   git log --since="YYYY-MM-DD" --until="YYYY-MM-DD" \
     --author="$GIT_USER" --name-only --pretty=format: \
     -- "help/quicksilver/**/*.md" | sort -u | grep -v '^$'
   ```

   d. これらの結果から、**現在のユーザーのコミットが日付範囲で実際にプレビューコンテンツを追加または変更したファイルにフィルター**。 候補ファイルごとに、ユーザーのコミットによってプレビューマーカーが導入されたかどうかを確認します。

   ```bash
   git log --since="YYYY-MM-DD" --until="YYYY-MM-DD" \
     --author="$GIT_USER" -p -- "<file>" | \
   grep -q '^\+.*class="preview"\|^\+.*{{highlighted-preview\|^\+.*highlighted information\|^\+.*not yet generally available'
   ```

   この正規表現が一致する場合にのみファイルを含めます（終了コード 0）。 これにより、他のユーザーによってプレビューのハイライトが追加されたファイルに対して、ユーザーが無関係な編集を行った場合の誤検出を回避できます。

   e. **上記の目次ルールに従って、**`product-announcements`**の下の任意のパス、**[&#x200B;除外されたパス ](#excluded-paths)**、および**&#x200B;目次/索引&#x200B;**ページを**&#x200B;省略します。\
   f. 結果の並べ替えリストを表示します。 リストされたファイルにプレビューのハイライトが表示されていないとユーザーが判断した場合は、編集を強制するのではなく、実行条件からファイルを削除して引き締めます。

2. **開始**\
   リストの&#x200B;**first**&#x200B;記事（またはユーザー名のパス）から始めるかどうかを尋ねます。

3. **記事ごと – 最初に表示し、OK**&#x200B;の後に編集
   - ファイルを読み込みます。
   - 編集を明確に提案する：**前/後の抜粋**、または変更される内容をフォーカスした差分スタイルの説明。
   - **ファイルを書き込む前に、明示的な承認（例：「OK」、「適用」、「はい」）を**&#x200B;待ってください。

4. **各ファイルの後**\
   **next**&#x200B;の記事に移動するかどうかを尋ねます（または停止/スキップ）。

## コンテンツルール（一般提供：コンテンツのプレビューがドキュメントになる）

**一般提供**&#x200B;のプレビュー強調表示を削除する場合、目標は次のとおりです。**プレビュー**&#x200B;用に文書化された動作を維持し、**古い「実稼動中」/「古いプロセス**」の分岐を削除し、**プレビュー専用のラベルとラッパーを削除**&#x200B;して、トピックがすべての顧客に対して最新として読み込まれるようにします。

### 平行ステップ

- 記事に手順（または番号付き項目）がある場合は、**「実稼動中」** （または同等の場合：現在の実稼動中/既存の実稼動中の動作） **および**、**平行** 「プレビュー中」 **（またはプレビュー環境）の順に指定します。**
   - **実稼動中の手順（古いパス）を**&#x200B;削除します。
   - **プレビュー中のステップの** substance **を**&#x200B;に保存しますが、**reword**&#x200B;は&#x200B;**プレビュー専用ではありません（「プレビュー中」、「プレビュー環境」などを削除します）。**&#x200B;リストの一貫性を保つために、番号を調整します。

構造があいまいな場合（明確な並列がない場合）、**stop**&#x200B;と両方の候補を表示します。ユーザーに、どのブロックを保持するかを尋ねます。

### 平行断面

- **セクション** （見出し+本文）が「実稼動環境」について&#x200B;**であり、「プレビュー環境」について** パラレルセクション ****&#x200B;がある場合&#x200B;**:**
   - **実稼動環境セクション（「置き換えられたコンテンツ」）を**&#x200B;削除します。
   - **プレビューセクションのコンテンツに**&#x200B;を置き換え、**削除** プレビュー環境の文言と&#x200B;**`class="preview"`**&#x200B;のラッパーを削除して、セクションが中立的な（GA対応）となるようにします。

### スニペットと記事の上部の通知

- **プレビュー専用のボイラープレート** ブロック（ハイライト表示されたコンテンツがプレビュー専用、高速リリース、サンドボックスなどであることを説明しているスパンまたはディブまたは段落）を削除します。 新しい機能を。
- ユーザーが別の通知を保持するように指示しない限り、**のみ**&#x200B;の目的がプレビューの可用性であるリンクまたは文章を削除します。

### HTML `class="preview"`

- 内部コンテンツ **（タグ内のmarkdown/HTML）を保持する&#x200B;**`class="preview"`**と**&#x200B;を使用するエレメントの&#x200B;**開始タグと終了タグ**&#x200B;を削除します。
- **`<span class="preview">`**&#x200B;と&#x200B;**`<div class="preview">`**&#x200B;の両方を処理し、他のタグ （例：**`<p class="preview">`**、**`<li class="preview">`**）で同じパターンが&#x200B;**目に見える** （コメントなし）本文コンテンツに表示される場合に処理します。
- リスト/テーブル構造を保持します。折れ曲がったリストまたは空の空白をアンラップ後に修正します。

### コメント付きセクション（HTMLのコメント）

- **ユーザーが明示的に何をすべきか**&#x200B;を示さない限り、**`<!-- … -->`**&#x200B;のHTML コメント **内のコンテンツを**&#x200B;削除、ラップ解除、またはその他&#x200B;**変更**&#x200B;しないでください（例：コメント全体を削除する、コメント内のプレビュークラスを削除する、GAのコメントを解除する）。
- コメント内にプレビュー関連のコンテンツまたは&#x200B;**`class="preview"`**&#x200B;が&#x200B;**のみ**&#x200B;表示される場合、**記事を確認する際に**&#x200B;を呼び出します。コメント内の内容を言い、**質問**&#x200B;をしてください。 **既定：コメント付きセクションは変更しないでください。**

### してはいけないこと

- このワークフローを&#x200B;**`product-announcements`** （リリースノートおよび関連）のパスで実行しないでください。インベントリでは、これらのパスを除外する必要があります。
- ユーザーが明示的にパスを含めるように求める場合を除き、**[除外されたパス](#excluded-paths)**&#x200B;にリストされているパスをインベントリまたは編集しないでください。
- **自動的に** コメント済み&#x200B;**（`<!-- … -->`）ブロックを削除または編集しないでください。上記の** コメント済みセクション **に従ってください。**
- この機能の可用性パターンについて&#x200B;**not**&#x200B;の場合は、「プレビュー」を削除しないでください（例：[ サンドボックス環境](·)を&#x200B;**製品名**&#x200B;として無関係なコンテキストでプレビューする）。判断して、不明かどうかを確認します。
- ユーザーが要求しない限り、`author:`または無関係なフロントマターを変更しないでください。
- **show → approve** ステップをスキップしないでください。

## 編集内容を提示する前に品質チェックを行う

- 合意された変更の範囲に残りの&#x200B;**`class="preview"`**&#x200B;はありません。
- 孤立した重複した見出しや壊れたステップ番号はありません。
- 概要の読み取り方が正しく&#x200B;**で、**&#x200B;はGAと矛盾しています（出荷済み機能の場合は「プレビューのみ」ではありません）。
