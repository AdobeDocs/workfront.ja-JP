# ミニ目次を追加

Markdown ファイルをスキャンし、直接小見出しを持つ各修飾見出しの下にミニ目次を挿入します。

## ワークフロー

1. ターゲットファイルを読み込みます。
2. `##` レベル以上のすべての見出しのうち、少なくとも1つの直接子見出し（1つ以上の`#` レベル）を持つものを特定します。
3. そのような親の見出しごとに、その子へのリンクの箇条書きリストのみを作成します。
4. 最初の子見出しの直前に、親の見出しの後に続く入門的なテキストの後ろにリストを挿入します。
5. ミニ目次がその位置に既に存在する場合は、そのセクションの現在の子見出しと比較します。 リストが古い（エントリが見つからない、追加のエントリがある、またはリンクが正しくない）場合は、更新されたリストに置き換えます。 すでに一致する場合は、スキップします。
6. 更新したファイルを書き込みます。

## 範囲

- **決して**&#x200B;さんが`#`記事タイトルの下にミニ目次を作成しません。 ミニ目次は`##`見出しより深い位置にのみ追加されます。
- `##`見出しには、`###`個の直属の子のリストが表示されます。
- `###`見出しには、`####`個の直属の子のリストが表示されます。
- より深いレベルで。

## リンク形式

リストの各エントリは、次の形式を使用します。

```
* [Heading text](#anchor)
```

### アンカー生成ルール

次のように、見出しテキストをアンカーに変換します。

1. すべてのテキストを小文字にします。
2. 文字、数字、スペース、またはハイフン以外の文字を削除します。
3. スペースをハイフンに置き換えます。
4. バックティックでラップされたコードの書式を削除します（テキストは内側に保持します）。

例：`### Create or edit a function` → `#create-or-edit-a-function`

## ネスティングルール

- 各リスト内の&#x200B;**直接子** （親より1 レベル深い）のみをリンクします。
- 同じリストに孫やより深い見出しを含めないでください。
- その子の見出し自体に子がある場合、その下に独自のミニ目次が挿入されます。

**構造の例：**

```
## Manage a package          ← parent: gets a list of ### headings
### Functions                ← child of ##, parent of ####: gets a list of #### headings
#### Create a function       ← child of ###
#### Delete a function       ← child of ###
### Variables                ← child of ##
### History                  ← child of ##
```

結果：

`## Manage a package`以下：

```
* [Functions](#functions)
* [Variables](#variables)
* [History](#history)
```

`### Functions`以下：

```
* [Create a function](#create-a-function)
* [Delete a function](#delete-a-function)
```

## プレースメント

最初の子見出しの直前にミニ目次リストを挿入します。 親見出しと最初の子見出しの間に入門テキストがある場合、リストはそのテキストの後に続き、最初の子見出しの上に直接配置されます。 ミニ目次リストの前後には、必ず空白行を含めます。

## スキップする内容

- `#`見出し（記事タイトル）：ミニ目次をここに追加しないでください。
- 1つのダイレクト子のみを持つ親の見出し：スキップ – 単一の項目リストではナビゲーション値は追加されません。
- 子ヘッダーのないセクション：スキップ。
